# 10.02. Системы мониторинга

- 1). Опишите основные плюсы и минусы pull и push систем мониторинга.
  
  PUSH
  - (+) отправление метрик в разные сервера
  - (+) гибкие настройки на агентах
  - (+) низкие расходы на транспорт UDP
  - (+) возможность отправлять метрики в назначенное время
  - (-) объем отправляемых данных может быть большой
  
  PULL
  - (+) какие агенты опрашивать регулируем на сервере, нет спамяших агентов.
  - (+) возможность использования прокси сервера, если нет прямого доступа к серверу
  - (+) упрощенная отладка
  - (-) единая точка отказа - в случае если сервер down, метрики не сладываются в БД.
  - (-) нагрузка на сервер увеличивается с ростом агентов

- 4).
  
  Перейдите в веб-интерфейс Chronograf (http://localhost:8888) и откройте вкладку Data explorer.

  Нажмите на кнопку Add a query
  Изучите вывод интерфейса и выберите БД telegraf.autogen
  В measurments выберите mem->host->telegraf_container_id , 
  а в fields выберите used_percent. Внизу появится график утилизации оперативной памяти в контейнере telegraf.
  Вверху вы можете увидеть запрос, аналогичный SQL-синтаксису. 
  Поэкспериментируйте с запросом, попробуйте изменить группировку и интервал наблюдений.
  Для выполнения задания приведите скриншот с отображением метрик утилизации места на диске 
  (disk->host->telegraf_container_id) из веб-интерфейса.
  
  - [disk_usage](images/10.02_3.PNG)
